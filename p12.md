# React Hooks pada Project CRM Paws & Care

## Deskripsi Project

Paws & Care merupakan sistem Customer Relationship Management (CRM) untuk klinik hewan yang digunakan untuk mengelola data customer (pemilik hewan), data hewan peliharaan, jadwal pemeriksaan, rekam medis, promosi, feedback pelanggan, data dokter dan staf, serta aktivitas operasional klinik. Dalam pengembangannya, React Hooks digunakan untuk mengelola state aplikasi, menjalankan proses otomatis setelah render, serta mengakses elemen tertentu secara langsung pada antarmuka aplikasi.

# A. useState

## What

useState digunakan untuk mengelola data yang dapat berubah pada antarmuka aplikasi CRM. Data tersebut meliputi nilai pencarian customer, data form appointment, data campaign promo, data feedback pelanggan, serta status modal dan filter.

## Why

useState diperlukan agar perubahan data dapat langsung diperbarui pada tampilan aplikasi tanpa perlu melakukan refresh halaman. Dengan demikian pengalaman pengguna menjadi lebih responsif dan interaktif.

## Who

Hook ini digunakan oleh admin klinik, dokter, dan staf yang melakukan pengelolaan data customer, hewan, jadwal pemeriksaan, promosi, maupun feedback pelanggan.

## When

State akan berubah ketika pengguna melakukan interaksi seperti:

* Mengetik pada kolom pencarian Customer CRM.
* Menambahkan appointment baru.
* Mengubah data pet owner.
* Membuat campaign promo.
* Mengirim feedback atau komplain.
* Melakukan filter data pada tabel.

## Where

Implementasi useState diterapkan pada beberapa halaman berikut:

### Customer CRM

Digunakan untuk menyimpan nilai pencarian customer dan filter data customer.

### Campaign Promo

Digunakan untuk menyimpan data form promosi seperti judul promo, deskripsi, dan periode promo.

### Feedback & Komplain

Digunakan untuk menyimpan data feedback, status komplain, dan hasil tindak lanjut.

### Pets

Digunakan untuk menyimpan data pencarian dan filter hewan peliharaan.

### Appointments

Digunakan untuk menyimpan data appointment dan status jadwal.

### Pet Owners

Digunakan untuk menyimpan data owner dan proses pencarian owner.

### Dokter & Staf

Digunakan untuk menyimpan data dokter, staf, dan proses pengelolaannya.

## How

Ketika pengguna melakukan perubahan data, React akan memperbarui state menggunakan fungsi setter dari useState. Setelah state berubah, React melakukan render ulang komponen sehingga tampilan selalu menampilkan data terbaru.

# B. useEffect

## What

useEffect digunakan untuk menjalankan proses tertentu setelah komponen berhasil dirender oleh React.

## Why

Hook ini diperlukan untuk mengambil data dari API atau database, memperbarui dashboard, serta melakukan sinkronisasi data ketika terjadi perubahan pada aplikasi.

## Who

Seluruh pengguna sistem mendapatkan manfaat dari useEffect karena data yang ditampilkan selalu diperbarui secara otomatis sesuai kondisi terbaru.

## When

useEffect dijalankan pada kondisi berikut:

* Saat halaman pertama kali dibuka.
* Saat data customer berubah.
* Saat data appointment berubah.
* Saat feedback baru ditambahkan.
* Saat campaign promo diperbarui.
* Saat statistik dashboard perlu diperbarui.

## Where

### Dashboard

Mengambil data statistik customer, transaksi, komplain, dan aktivitas sistem.

### Customer CRM

Mengambil daftar customer ketika halaman dibuka.

### Feedback & Komplain

Mengambil data komplain dan feedback terbaru.

### Appointments

Mengambil daftar jadwal pemeriksaan dan melakukan pembaruan data ketika terjadi perubahan.

### Campaign Promo

Mengambil data promosi yang aktif dan menampilkan status promosi terkini.

## How

Saat komponen dimuat, useEffect akan dijalankan untuk mengambil data yang diperlukan. Jika dependency pada useEffect berubah, React akan menjalankan kembali proses tersebut sehingga data yang ditampilkan tetap konsisten dan selalu terbaru.

# C. useRef

## What

useRef digunakan untuk menyimpan referensi elemen DOM atau nilai tertentu tanpa menyebabkan render ulang komponen.

## Why

Tidak semua perubahan data perlu ditampilkan pada antarmuka. Pada kondisi tertentu aplikasi hanya membutuhkan akses langsung ke elemen tertentu sehingga useRef lebih efisien dibandingkan useState.

## Who

Admin dan staf klinik yang sering melakukan input data akan merasakan manfaat fitur ini karena membantu mempercepat proses penggunaan aplikasi.

## When

useRef digunakan ketika:

* Sistem perlu memberikan fokus otomatis pada kolom pencarian.
* Sistem perlu mengakses form tertentu secara langsung.
* Sistem perlu menyimpan nilai sementara tanpa render ulang.

## Where

### Customer CRM

Digunakan untuk memberikan fokus otomatis pada kolom pencarian customer.

### Feedback & Komplain

Digunakan untuk memberikan fokus pada form pengisian komplain.

### Campaign Promo

Digunakan untuk memberikan fokus pada form pembuatan promo baru.

### Appointments

Digunakan untuk mengakses form appointment saat proses penjadwalan.

## How

React menyimpan referensi elemen ke dalam objek ref. Melalui properti `current`, aplikasi dapat mengakses elemen tersebut secara langsung tanpa memicu render ulang komponen. Cara ini membuat performa aplikasi lebih efisien terutama pada proses yang tidak memerlukan pembaruan tampilan.

---

# Kesimpulan

Pada sistem CRM Paws & Care, useState digunakan untuk mengelola data yang berubah akibat interaksi pengguna, useEffect digunakan untuk mengambil dan memperbarui data secara otomatis setelah proses render, sedangkan useRef digunakan untuk mengakses elemen tertentu secara langsung tanpa menyebabkan render ulang komponen. Ketiga React Hooks tersebut berperan penting dalam membangun aplikasi CRM yang responsif, efisien, dan mudah digunakan oleh admin, dokter, maupun staf klinik.
