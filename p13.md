# PENAMBAHAN FITUR BARU - CRM PETCARE

## DESKRIPSI

Tambahkan fitur baru ke dalam project CRM PetCare yang sudah ada.

JANGAN mengubah fitur yang sudah berjalan.

JANGAN mengubah struktur dashboard utama.

JANGAN mengubah branding aplikasi.

JANGAN mengubah UI yang sudah ada.

Lakukan hanya penambahan fitur autentikasi dan manajemen user menggunakan Supabase.

---

# FITUR BARU : SUPABASE AUTHENTICATION

Saat ini sistem login masih menggunakan data dummy, mock data, fake API, atau JSON lokal.

Ganti mekanisme tersebut dengan Supabase Authentication.

Login harus langsung terhubung ke Supabase.

---

# FITUR BARU : LOGIN SUPABASE

Tambahkan login menggunakan Supabase Auth.

Field:

* Email
* Password

Validasi:

* Email wajib diisi
* Password wajib diisi

Tampilkan:

* Loading State
* Error Message
* Success Login

Jika berhasil login:

* Simpan session user
* Redirect ke Dashboard CRM PetCare

Jika gagal:

* Tampilkan pesan error yang sesuai

---

# FITUR BARU : REGISTER PAGE

Tambahkan halaman registrasi baru.

Nama halaman:

Register Page

Field:

* Nama Lengkap
* Email
* Nomor Telepon
* Password
* Konfirmasi Password

Validasi:

* Semua field wajib diisi
* Email valid
* Password minimal 8 karakter
* Password dan konfirmasi password harus sama

---

# FITUR BARU : REGISTRASI KE SUPABASE

Saat registrasi berhasil:

1. Buat akun pada Supabase Authentication.
2. Simpan data profil user ke tabel users.
3. Tampilkan notifikasi sukses.

Contoh:

"Akun berhasil dibuat. Silakan login menggunakan akun Anda."

User yang baru mendaftar harus langsung dapat login.

---

# FITUR BARU : DATABASE USERS

Tambahkan tabel users pada Supabase.

Kolom:

* id
* auth_user_id
* full_name
* email
* phone_number
* role
* created_at
* updated_at

Role default:

customer

---

# FITUR BARU : CRUD USER

Tambahkan modul User Management untuk Admin.

Admin dapat:

## Create User

Menambahkan user baru.

## Read User

Melihat seluruh daftar user.

Tampilkan:

* Nama
* Email
* Nomor Telepon
* Role

## Update User

Mengubah:

* Nama
* Nomor Telepon
* Role

## Delete User

Menghapus user.

Tambahkan dialog konfirmasi sebelum delete.

---

# FITUR BARU : ROLE MANAGEMENT

Role:

* admin
* staff
* customer

---

## Admin

Dapat:

* CRUD User
* CRUD Customer
* CRUD Pet
* CRUD Appointment
* CRUD Service

---

## Staff

Dapat:

* Melihat Customer
* Melihat Pet
* Mengelola Appointment
* Mengelola Service

Tidak dapat mengelola User.

---

## Customer

Dapat:

* Melihat profil sendiri
* Mengubah profil sendiri
* Membuat appointment
* Melihat riwayat layanan

---

# FITUR BARU : PROFILE PAGE

Tambahkan halaman Profile.

Data berasal dari Supabase.

Tampilkan:

* Nama Lengkap
* Email
* Nomor Telepon
* Role

User dapat mengubah:

* Nama Lengkap
* Nomor Telepon

Email hanya dapat dibaca.

---

# FITUR BARU : SESSION MANAGEMENT

Gunakan Supabase Session.

Saat halaman direfresh:

* User tetap login jika session masih aktif.

Saat logout:

* Hapus session Supabase.
* Redirect ke Login Page.

---

# FITUR BARU : SECURITY

Gunakan Row Level Security (RLS).

Policy:

* Customer hanya dapat mengakses datanya sendiri.
* Staff memiliki akses sesuai tugas.
* Admin memiliki akses penuh.

Jangan menonaktifkan RLS.

---
